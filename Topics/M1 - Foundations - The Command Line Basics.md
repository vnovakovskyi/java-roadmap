# The Command Line Basics

> [!info] Topic
> **Area:** Foundations · **Module:** 1 · **Priority:** must · **Est:** 3h
> **Why this matters:** Real Java work — compiling, running, building, Git — happens at the command
> line. You don't need to be an expert, but you must be comfortable navigating and running commands.

## Learning Objectives
- Navigate the filesystem (list, change directory, see where you are).
- Create folders/files and inspect them from the terminal.
- Run a command and read its output, including errors.

## Learn
- [ ] Read a short shell primer for your OS (macOS/Linux: bash/zsh; Windows: PowerShell).
- [ ] Learn these commands: `pwd`, `ls`/`dir`, `cd`, `mkdir`, `cat`/`type`, clearing the screen.

## Concepts
- [ ] The shell has a **current directory**; paths are relative to it unless absolute.
- [ ] `.` = here, `..` = up one level, `~` = home directory.
- [ ] Commands return output and an **exit status** (0 = success); errors print to the screen.
- [ ] Tab-completion and the up-arrow (command history) save enormous time.

## Practice
**Goal:** Compile and run Java from the terminal — no IDE.
**Steps:**
- [ ] Make a folder, `cd` into it, and create a `Hello.java` with your editor.
- [ ] Compile it: `javac Hello.java` (note the new `Hello.class`).
- [ ] Run it: `java Hello`.
- [ ] List the directory before and after compiling and notice what changed.
